<!-- ============================================================
  書き方のガイド

  ■ 見出し構造
    ## 文書タイトル
    ### ＜カテゴリ名＞           ... Category.name
    #### キーワード: 上位要求 ID  ... Requirement（keywords + id）
    ##### 下位要求 ID            ... SubRequirement（2階層時のみ）
    ###### ＜仕様グループ名＞     ... SpecificationGroup.name
    - □□□ 仕様ID: 仕様テキスト   ... Specification

  ■ 仕様ラベル（□□□）
    □□□ = 未レビュー     [false, false, false]
    ■□□ = レビュー済み   [true, false, false]
    ■■□ = 実装済み       [true, true, false]
    ■■■ = テスト済み     [true, true, true]

  ■ ID命名の推奨パターン
    カテゴリ:   C01, C02, ...
    要求:       R01, R02, ...
    下位要求:   R01-01, R01-02, ...（親要求IDをプレフィクス）
    仕様:       S01-01-01, S01-01-02, ...

  ■ 仕様グループ名
    全角山括弧＜＞で囲む。要求の目的語と動詞で区切った
    観点ごとに分類する（例: ＜画面表示＞ ＜データ処理＞ ＜エラー処理＞）

  ■ パターンの選択
    1階層: 要求がシンプルで分割不要 → specificationGroups を直接使用
    2階層: 要求が複合的で分割が必要 → requirementGroups 経由で下位要求に分割

  ■ 対応するCanonical JSON Schema
    schema/v1/document.schema.json

============================================================ -->
